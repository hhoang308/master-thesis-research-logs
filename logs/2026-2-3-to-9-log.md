#### debugging tools for windows
- GFlags
    - includes the funtionality of PageHeap (pageheap.exe), a tools that enables heap allocation monitoring in Windows and enables features that reserve memory at the boundary of each allocation to detect attempts to access memory beyond the allocation.
    - 2 options of pageheap: standard heap verification and full-page heap verification. 
- WinDBG
    - `bm /a jp2klib!* "..."` -> bm: breakpoint match, /a: all matched functions, jp2klib!*: pattern, `...`: command sequence automatically execute when breapoint hits.
    - `k L5` -> prints first 5 lines of call stack.
    - `dc esp L8` -> prints in the form of double word and char, `esp` is register for stack pointer => prints first 8 elements (4 * 8 = 32 bytes), which is the parameter.
    - `pt` -> step to next return, which means, continue until meet the return command of this function.
