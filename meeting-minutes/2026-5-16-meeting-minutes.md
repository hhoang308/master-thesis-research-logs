Xây dựng văn phạm để build được pattern
- tìm lệnh để xoá/thêm/tương tác với object 
- kiểu dữ liệu của các object
Các tình huống
- chuỗi tham chiếu
- tham chiếu xong rồi xoá object
- tạo các object đặc biệt
Thay đổi kiểu dữ liệu (tấn công can nhiễu tham số, paremeter tampering): giá trị biên, int -> float, string, escape character, 

Khái quát hoá thành 2 pattern trong nautilus để generate được các pattern này

Kiểm thử hộp xám bằng nautilus
Không giới hạn ở PDF brower/viewer hỗ trợ JS Enginge, mà mở rộng ra toàn bộ định dạng pdf.

Các việc cần làm:
1. Tìm và phân tích CVE tiếp.
2. Tìm 1-2 lệnh thêm/sửa/xoá/tương tác với 1-2 object trong pdf và các kiểu dữ liệu tham số trong các object đó.
3. Tìm văn phạm để tạo file pdf.
